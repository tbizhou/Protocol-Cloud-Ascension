# 游戏策划案：《云端飞升协议》(Protocol: Cloud Ascension)

**版本:** 1.0  
**平台:** Web / H5 (移动端 & PC端适配)  
**类型:** 放置挂机 (Idle) + 策略构建 (DBG) + 赛博修仙 MUD  

---

## 1. 核心概述 (Executive Summary)

### 1.1 游戏简介
《云端飞升协议》是一款极简主义的Web端放置游戏。游戏将东方修仙体系（境界、渡劫、丹药）与赛博朋克题材（权限、防火墙、硬件）深度结合。玩家扮演一名试图通过不断优化自身代码与硬件，将意识上传至“零号根服务器”的赛博格游荡者。

### 1.2 核心体验 (Slogan)
> **“肉体苦弱，机械飞升；编译灵魂，直达根服。”**

### 1.3 目标用户
* 程序员 / 技术爱好者
* 修仙小说读者
* 放置挂机游戏（Incremental Games）玩家
* 上班族（适合作为浏览器侧边栏的“摸鱼”游戏）

---

## 2. 世界观设定：虚实二象性

### 2.1 物质层 (The Hardware Layer)
* **环境**：霓虹贫民窟，资源枯竭，名为“旧扇区”。
* **生存法则**：肉体只是载体，唯有算力是硬通货。
* **主要势力**：
    * **天元重工 (Tianyuan Industries)**：垄断高端义体硬件（炼体宗门）。
    * **玄微软件 (Xuanwei Soft)**：垄断算法与带宽（符箓/法修宗门）。

### 2.2 数据层 (The Digital Layer)
* **太虚幻境 (The Grid)**：由算力构成的维度，代码即法术。
* **飞升 (Ascension)**：突破物理限制，将意识数据化，上传至传说中的“根服务器 (Root Server)”，获得永恒权限。
* **天劫 (The Purge)**：网络监察AI会定期扫描全网。当个体权限（境界）过高时，系统会降下“杀毒程序”试图抹杀异常数据。

---

## 3. 核心玩法循环 (Core Loop)

1.  **挂机 (Idle)**：后台运行挖矿脚本，获取基础资源“算力”与“比特金”。
2.  **构建 (Build)**：消耗比特金升级硬件（肉身），扩大内存与散热上限。
3.  **编程 (Code)**：消耗算力编写/合成战斗脚本（功法），配置技能循环。
4.  **攻坚 (Breach)**：挑战副本或渡劫BOSS（防火墙），获取更高系统权限（境界）。

---

## 4. 详细系统设计

### 4.1 境界系统 (Permissions Hierarchy)
修仙境界对应系统权限，决定了玩家在网络世界的层级。

| 修仙境界 | 赛博称谓 | 权限描述 | 核心玩法 |
| :--- | :--- | :--- | :--- |
| **练气** | **脚本小子 (Script Kiddie)** | User (只读) | 基础挂机，手动点击挖矿。 |
| **筑基** | **全栈架构 (Full Stack)** | Admin (执行) | 开启“内网洞府”，自动化产出资源。 |
| **金丹** | **内核编译 (Kernel Build)** | System (核心) | 获得“逻辑核心”，开启高级战斗脚本构建。 |
| **元婴** | **虚拟实体 (Virtual Entity)** | Daemon (守护进程) | 意识脱离硬件，多线程探索暗网副本。 |
| **化神** | **分布式神 (Distributed Mind)** | Cluster (集群) | 跨服算力借用，公会战（DDoS攻击）。 |
| **大乘** | **根用户 (Root User)** | Root (最高权限) | 修改世界规则，开启多周目（转生）。 |

### 4.2 硬件与肉身 (Hardware Attributes)
硬件决定基础属性，是承载代码的容器。

* **CPU (灵根)**：决定 **算力获取效率** (Hash/s)。
* **内存 RAM (识海)**：决定 **脚本负载上限** (Memory Capacity)。越强大的功法占用的内存空间越大。
* **硬盘 (丹田)**：决定 **算力储存上限**。
* **散热系统 (经脉)**：决定 **过热阈值**。战斗和高速运转会产生热量，过热会导致“走火入魔”（Debuff）。

### 4.3 战斗系统：可视化代码流
战斗采用**自动回合制 + 策略构建 (DBG)**。

* **界面表现**：终端窗口风格，红蓝代码滚动对抗。
* **机制**：
    * **脚本插槽 (Script Deck)**：玩家拥有有限的内存槽位（例如 32GB）。
    * **配置策略**：玩家需在战前配置好脚本的执行顺序。
* **技能示例 (JSON风格)**：
    * `Ping_Flood()`: 基础攻击，低耗能。
    * `Sudo_Kill()`: 高伤害，消耗大量热量，需root权限概率触发。
    * `Firewall_Patch()`: 增加临时护盾。
    * `While(True)_Heal()`: 持续回血，容易导致过热。

### 4.4 渡劫系统 (Boss Fight)
* **触发条件**：算力修满，且散热系统正常。
* **Boss**：**网监杀毒卫士 (The Enforcer)**。
* **玩法**：DPS Check（伤害检测）。玩家必须在散热值爆表（被检测封号）之前，利用爆发性脚本击穿防火墙。
* **失败惩罚**：境界回退，丢失部分未保存的代码碎片。

---

## 5. 资源与经济

* **算力 (Hashrate/Qi)**：核心经验值，用于升级软件版本（境界）。
* **比特金 (Credits)**：通用货币，用于购买硬件。
* **Bug碎片 (Karma)**：特殊掉落物，用于合成或改良脚本（炼丹）。
* **散热剂 (Coolant)**：消耗品，用于渡劫时强制降温。

---

## 6. UI/UX 与美术风格

### 6.1 视觉风格
* **终端机美学 (Terminal Aesthetics)**：黑底绿字/粉字，等宽字体。
* **故障艺术 (Glitch Art)**：受到伤害或暴击时，界面出现噪点和撕裂。
* **CRT滤镜**：模拟老式显示器的扫描线效果。

### 6.2 界面布局 (IDE风格)
* **左侧 (Explorer)**：导航菜单（状态、背包、黑市、副本）。
* **中间 (Editor)**：
    * *非战斗*：滚动的修炼日志 (`[System] Absorbing 1024 Hash...`)。
    * *战斗中*：战斗代码流。
* **右侧 (Debug)**：可视化的硬件状态图（人体经络与电路板的结合）。

---

## 7. 技术实现建议 (Web)

* **前端框架**：Vue 3 或 React (组件化开发)。
* **UI库**：Tailwind CSS (快速实现暗黑赛博风格)。
* **状态管理**：Pinia / Redux (存储复杂的玩家状态与背包)。
* **后台计算**：Web Workers (确保挂机计算不阻塞主线程UI)。
* **数据持久化**：
    * `LocalStorage` (自动保存)。
    * `Export Base64 String` (允许玩家导出存档代码，方便分享)。
* **彩蛋**：在浏览器 F12 Console 中输出隐藏剧情或彩蛋礼包码。

## 8. 扩展规划 (Roadmap)

* **Phase 1**：基础挂机循环，练气到元婴内容，单机模式。
* **Phase 2**：加入 Roguelike 副本（暗网探索），随机事件系统。
* **Phase 3**：轻度社交（局域网宗门），WebSocket 实现简单的聊天与排行榜。
