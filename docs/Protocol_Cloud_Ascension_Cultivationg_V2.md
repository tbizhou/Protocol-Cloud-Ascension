# 游戏策划案：《云端飞升协议》 (Protocol: Cloud Ascension)

**版本:** 2.0 (系统深度版)  
**更新重点:** 煅体/修灵双路线设计、隐藏机魂双修、PVE运行时战斗系统  
**平台:** Web / H5  

---

## 1. 核心概述

### 1.1 游戏简介
一款融合了赛博朋克美学与东方修仙哲学的Web端放置RPG。玩家需要在“升级硬件（肉身成圣）”与“优化代码（羽化登仙）”之间做出抉择，最终突破物理层与逻辑层的界限，达成“根服务器”的飞升。

### 1.2 核心循环
1.  **资源积累**：挂机获取算力（时间）与比特金（金钱）。
2.  **路线抉择**：投入资金升级硬件（煅体），或投入时间研习算法（修灵）。
3.  **战斗配置**：根据路线特性，构建“指令堆栈”。
4.  **运行时攻坚**：在PVE时间轴战斗中击败电子妖兽与防火墙。

---

## 2. 修炼路线分歧 (Cultivation Paths)

游戏不设职业选项，流派完全由玩家的资源投入方向决定。

### 2.1 路线一：煅体流 (The Hardware Path)
> *“只要算力足够大，砖头也能飞上天。”*

* **核心资源**：**比特金 (Credits)**。重氪金（游戏内货币），轻时间。
* **晋升方式**：购买更昂贵的 CPU、机箱、液氮冷却、核能电池。
* **核心属性**：
    * **结构完整性 (HP)**：极高的血量，能硬抗伤害。
    * **吞吐量 (Throughput)**：能量恢复极快，支持高频技能释放。
    * **散热阈值 (Cooling)**：允许极其暴力的“超频”爆发。
* **战斗风格**：**暴力破解 / 物理碰撞**。
    * 利用巨大的指令块直接撞碎敌人的攻击。
    * 擅长反伤（物理防火墙）和单次高额爆发。
* **缺陷**：解决复杂逻辑异常（Debuff）的能力较弱，维护成本高昂。

### 2.2 路线二：修灵流 (The Code Path)
> *“万物皆虚，万事皆允。修改代码，即修改现实。”*

* **核心资源**：**算力 (Hashrate) & 挂机时间**。重积累，轻金钱。
* **晋升方式**：通过挂机冥想（Deep Learning）重构代码、压缩算法、寻找系统漏洞。
* **核心属性**：
    * **逻辑深度 (Logic Depth)**：决定技能的优先级（判定权）。
    * **多线程 (Threads)**：决定一回合能并发执行多少个动作。
    * **编译速度 (Cast Speed)**：极短的技能冷却。
* **战斗风格**：**代码注入 / 逻辑规避**。
    * 指令块细小而密集，能穿过敌人攻击的缝隙直击核心。
    * 擅长控制（Freeze）、中毒（Virus）和篡改敌人属性。
* **缺陷**：身板极脆（低HP），一旦被击中容易暴毙；极易过热。

### 2.3 隐藏路线：机魂双修 (Bio-Digital Fusion)
> *“肉身是舟，代码是帆。”*

这是游戏的最强形态，但开启极为困难，且维护难度极高。

#### 2.3.1 开启条件 (隐藏)
玩家必须触发 **"系统崩溃 (Kernel Panic)"** 事件：
1.  **双项圆满**：硬件等级与代码境界完全持平（例如同为筑基期圆满）。
2.  **濒死重构**：在任意 BOSS 战中，HP 被打至 0 的同时，依靠“自动重启”类代码复活，且此时 **散热值 > 95%**。
3.  **结果**：战斗结束后触发蓝屏死机特效，控制台询问 `INITIATE FUSION? [Y/N]`。

#### 2.3.2 双修机制：同步率 (Sync Rate)
双修引入了 **“同步率”** 这一惩罚/奖励乘区。
* **动态平衡**：玩家必须交替提升硬件和代码。如果一方过强，同步率下降。
* **低同步率惩罚**：
    * *硬件 > 代码*：**硬件空转**（攻击未命中率大幅上升）。
    * *代码 > 硬件*：**过热自爆**（每回合受到真实伤害反噬）。
* **高同步率奖励**：获得专属技能系（如 `Hardware_Interrupt`），享受双倍属性加成。

## 3. PVE 战斗系统：运行时 (The Runtime)

战斗不再是简单的你一拳我一脚，而是基于 **时间轴 (Timeline)** 的攻防博弈。

### 3.1 视觉表现
* **顶部**：一条从左向右流动的进度条（时间轴）。
* **中间**：主要显示区。
    * **红色块 (Exceptions)**：敌人抛出的攻击/异常。
    * **蓝色块 (Instructions)**：玩家发出的指令/技能。
* **底部**：玩家的 **内存堆栈 (Memory Stack)**，显示当前配置的技能循环。

### 3.2 战斗流程
1.  **编译期 (Pre-fight)**：玩家配置技能顺序（例如：`护盾 -> 病毒 -> 重击`）。
2.  **运行期 (Runtime)**：战斗开始，时间轴流动。
    * 敌人在特定时间点生成 **红色块**。
    * 玩家按顺序生成 **蓝色块**。
3.  **碰撞判定 (Resolution)**：当时间轴扫过色块时：

| 情况 | 煅体流 (硬件) 表现 | 修灵流 (代码) 表现 |
| :--- | :--- | :--- |
| **面对攻击** | **[撞碎]**：你的蓝色块巨大且坚硬，直接撞碎红块，抵消伤害并造成反震。 | **[穿梭]**：你的蓝色块细小，从红块的间隙（漏洞）钻过去，无视防御直接攻击BOSS本体。 |
| **造成伤害** | **[重锤]**：造成单次巨额物理伤害。 | **[多段]**：一瞬间造成10次小额伤害，触发多次特效。 |
| **特殊应对** | 强行使用 `Block` 技能暂停时间轴。 | 使用 `Rewind` 技能回退时间轴，重置战局。 |

### 3.3 关卡与BOSS设计

* **普通怪 (Daemons)**：单纯的数值怪，任何流派达标即可通过。
* **精英怪 (Logic Locks)**：
    * **物理锁**：血量极高，每回合回血。修灵流需注入“禁疗病毒”，煅体流需伤害碾压。
    * **逻辑锁**：物理免疫，必须用代码解谜（按特定顺序攻击）。煅体流需要消耗昂贵的一次性道具“破译器”才能通过。
* **最终BOSS (The Firewall)**：
    * 拥有 **[逻辑护盾]** (只能被代码伤害) + **[物理内核]** (只能被硬件伤害)。
    * **双修优势**：双修玩家可以无缝切换伤害类型，效率最高。
    * **单修策略**：单修玩家必须把属性练到极致（力大砖飞 或 极致解构）才能强行突破。

## 4. 资源经济系统 (Economy)

| 资源名称 | 获取途径 | 主要用途 | 对应路线 |
| :--- | :--- | :--- | :--- |
| **比特金 (Credits)** | 悬赏任务、黑市交易 | 购买 CPU、显卡、散热液 | **煅体流 (核心)** |
| **算力 (Hashrate)** | 挂机挖矿 (自动增长) | 升级软件版本、编写脚本 | **修灵流 (核心)** |
| **热量 (Heat)** | 战斗释放技能产生 | 限制战斗时长，满值后过热 | **通用限制** |
| **Bug碎片** | 击杀怪物掉落 | 炼丹（合成特殊脚本/强化硬件） | **通用材料** |

## 5. UI交互设计建议

* **主界面 (Dashboard)**：
    * 左侧显示 **[机体状态]**（ASCII 人体图）。若侧重煅体，身体部位高亮金属光泽；若侧重修灵，头部散发数据光环。
    * 右侧显示 **[运行日志]**。
* **战斗界面 (Runtime)**：
    * 模仿 **Chrome Network 瀑布流** 或 **音频编辑软件音轨** 的视觉风格。
    * 打击感通过屏幕震动（Glitch效果）和代码报错红字来体现。

---

## 6. 开发优先级 (Roadmap)

1.  **Phase 1 (基础)**：实现“挂机产出资源”与“基础属性购买”。
2.  **Phase 2 (战斗)**：实现“时间轴”UI，以及基础的“碰撞/穿梭”判定逻辑。
3.  **Phase 3 (分歧)**：实装两套不同的技能树，调整数值平衡。
4.  **Phase 4 (双修)**：加入隐藏触发条件和同步率机制。
