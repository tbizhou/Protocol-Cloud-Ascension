# 基础挂机升级系统：从脚本小子到内核金丹

**项目名称:** 云端飞升协议 (Protocol: Cloud Ascension)  
**版本:** 1.0 (Progression)  
**范围:** 练气 (Script Kiddie) $\to$ 筑基 (Full Stack) $\to$ 金丹 (Kernel)  
**核心循环:** 资源获取 $\to$ 瓶颈突破 $\to$ 境界提升

---

## 1. 核心资源循环模型 (The Loop)

挂机系统的本质是“资源转化效率”的博弈。在本作中，资源流向与修仙概念深度绑定：

1.  **算力 (Hashrate / Qi)** $\to$ **[修灵经验]**：用于提升软件版本（境界），优化代码。
2.  **比特金 (Credits)** $\to$ **[煅体资金]**：用于购买硬件装备，强化肉身。
3.  **热量 (Heat)** $\to$ **[限制阈值]**：挂机效率的惩罚机制，过热会降低产出。

### 1.1 基础挂机公式

$$
\text{产出效率} = (\text{CPU基准} \times \text{算法倍率}) \times (1 - \text{过热惩罚系数})
$$

## 2. 阶段一：练气期 (Script Kiddie)

**代号**：脚本小子  
**目标**：从手动点击过渡到自动化脚本。  
**核心体验**：解决散热问题，积累第一桶金。

### 2.1 挂机机制进化
* **Lv.1 - Lv.3 (手动)**：
    * 玩家需点击屏幕上的 `[RUN]` 按钮手动运行代码。
    * 每次点击获得少量算力。
* **Lv.4 (Hello World)**：
    * 获得第一个自动循环脚本，每秒自动运行一次。
    * 解锁 **[进程管理]** 面板。
* **Lv.5 - Lv.9 (多线程)**：
    * 开启多个后台进程（挂机线程），成倍增加产出。
    * **瓶颈出现**：**风冷散热**。随着线程增加，热量飙升，效率开始下降（Throttling）。

### 2.2 流派分歧初现
虽然未正式转职，但资源投入决定了早期优势：

| 关注点 | 煅体流 (Hardware Focus) | 修灵流 (Software Focus) |
| :--- | :--- | :--- |
| **资金用途** | 购买 **[强力风扇]** 和 **[独立显卡]** | 购买 **[云服务器时长]** 和 **[付费API]** |
| **挂机优势** | 散热好，能长时间稳定运行（细水长流）。 | 爆发高，但易过热死机，需经常上线重启（短时高效）。 |

### 2.3 突破事件：筑基防火墙 (The Firewall)
当练气圆满 (Lv.10) 时，算力槽满但无法升级。
* **事件**：系统提示 `Permission Denied`。
* **条件**：制造 **[筑基脚本 (Root Exploit)]**，并击败守门 BOSS (Port 21)。

## 3. 阶段二：筑基期 (Full Stack)

**代号**：全栈架构  
**目标**：搭建属于自己的“服务器洞府”。  
**核心体验**：硬件与软件的模块化管理，流派彻底分化。

### 3.1 系统解锁：洞府 (Server Room)
界面从单一终端变为可视化的机房。
* **硬件槽位 (Hardware Rack)**：插入 CPU、RAM、SSD。
* **软件容器 (Docker Containers)**：加载不同的挂机算法镜像。

### 3.2 挂机路线分化 (Hard Split)

#### A. 煅体流：矿机阵列 (Mining Rig)
* **核心逻辑**：**算力换金钱**。
    * 挂机产出的不再主要是“经验”，而是大量的“比特金”。
* **升级方式**：**堆料 (Stacking)**。
    * 购买昂贵的 **[液氮冷却]** 和 **[军用级机箱]**。
    * 不断更换更高制程的芯片 (14nm $\to$ 7nm $\to$ 3nm)。
* **筑基特性**：**[物理硬化]** —— 挂机时长越久，血量上限越高（硬件磨合）。

#### B. 修灵流：分布式计算 (Distributed Computing)
* **核心逻辑**：**算力换算力**。
    * 将所有算力投入到“自我迭代”中。
    * 挂机产出大量的“算法碎片”，用于合成高级技能。
* **升级方式**：**重构 (Refactoring)**。
    * 消耗算力重写底层代码，减少体积，提高执行速度。
* **筑基特性**：**[逻辑压缩]** —— 挂机时长越久，技能冷却缩减 (CDR) 越高。

## 4. 阶段三：金丹期 (Kernel Build)

**代号**：内核编译  
**目标**：将自身编译为一个独立的“核心”。  
**核心体验**：质变。

### 4.1 结丹机制 (Compiling)
这是一个漫长的挂机过程（类似闭关）。玩家需选择 **[一颗芯片]** + **[一套操作系统]** 进行融合编译。
* **硬件品质** $\to$ 金丹承载力 (HP/MP)。
* **代码品质** $\to$ 金丹品级 (一品至九品，决定攻击成长)。

### 4.2 专属挂机功能

#### A. 煅体流：核能反应堆 (Fusion Core)
* **变化**：体内生成反应堆，不再依赖外部电源。
* **挂机模式**：**[充能模式]**。
    * 离线时积攒“能量电荷”。
    * 上线后一键释放，造成全屏秒杀伤害（方便快速推图）。
    * 视觉特效：机体出现 RGB 呼吸灯光污染。

#### B. 修灵流：虚拟人格 (AI Persona)
* **变化**：意识独立于肉体，数据实体化。
* **挂机模式**：**[多线程神游]**。
    * 允许同时挂两个副本：主线程挂机当前层，子线程去低级副本自动刷材料 (Autoloot)。
    * 效率最大化：同时获取经验和材料。

## 5. 隐藏路线：机魂双修 (Dual Stack)

**开启条件**：在筑基期，强行保持 `硬件评分 == 软件评分`，并触发瀕死重构。

### 5.1 独特的挂机机制：同步率调优 (Sync Tuning)
双修玩家没有“无脑挂机”的权利，必须玩 **[平衡小游戏]**。

* **界面**：示波器显示两条正弦波（硬件波 & 软件波）。
* **操作**：
    * **硬件波过高 (电压过载)**：手动扣除比特金，注入冷却剂。
    * **软件波过高 (逻辑死锁)**：手动扣除算力，进行垃圾回收 (GC)。
* **奖励/惩罚**：
    * **完全重合**：挂机收益 **x300%**。
    * **偏离 > 20%**：挂机收益 **x10%**（严重惩罚）。

### 5.2 金丹特效：量子纠缠态 (Quantum State)
* **产物**：结出的不是金丹，而是 **[量子比特球]**。
* **收益**：挂机产出“混合态资源包”。打开后随机获得比特金或算力，且有概率开出 **[双修专属技能书]**。

### 7. 数值成长曲线参考 (Progression Curve)


| 境界阶段 | 平均挂机时长 (升至下一级) | 核心资源阈值 | 解锁核心功能 |
| :--- | :--- | :--- | :--- |
| **练气 (Lv.1-9)** | 10分钟 $\to$ 2小时 | 1,024 Hash | 自动脚本、多线程 |
| **筑基 (Lv.1-9)** | 4小时 $\to$ 2天 | 100万 Credits / 100万 Hash | 洞府机房、流派特性 |
| **金丹 (Lv.1-9)** | 3天 $\to$ 1周 | 1亿 Credits / 1亿 Hash | 核能爆发、多线程挂机 |

**设计意图：**
1.  **练气期**：短平快，快速给予正反馈，让玩家熟悉系统。
2.  **筑基期**：拉长战线，强迫玩家在“搞钱（煅体）”和“搞技术（修灵）”中二选一，确立流派玩法。
3.  **双修路线**：高风险高回报，通过强制活跃（需要频繁上线手动调优）来平衡其 300% 的数值优势，避免纯挂机导致平衡崩坏。
